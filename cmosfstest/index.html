TommyOS

Welcome to the official TommyOS page!


TommyOS is a custom x86/16 bit operating system fully coded in assembly.

The page below describes the installation progress, system requirements, and an entire documentation page on how to make your own programs.




Installation and download

The file above is a virtual floppy disk image. To create a bootable floppy disk with TommyOS, first get a 1.44MB floppy disk.

Then use a disk imaging tool like Roadkil's Disk Image and burn the image to the floppy.

Once you have done that, pop the floppy into an IDE floppy drive. This is important.

TommyOS runs on what are now legacy systems. Most modern x64 systems don't support booting from floppy disks or 16 bit assembly anymore.




System requirements

TommyOS requires an x86 CPU, primarily a Pentium, but it doesn't matter.

64 kilobytes or more of RAM,

PS/2 keyboard,

Built in speaker.




Documentation

Quick start

To get started with TommyOS as quickly as possible, just burn the image to a floppy and pop it in!

Or, just run with any virtualization software like Qemu, VMware, or VirtualBox.

On boot into the command line interface, type 'help' for a list of commands.

Then type 'dir', and view all the preloaded files.

In TommyOS, you can run .bin files by just typing their name. Each bin file gets one param, like for WRITE.BIN you type 'write test.txt', and you will create and start editing a file called test.txt.

You can delete, rename, copy, and see the size of any almost any file in TommyOS.

You can also view PCX images in the command line.




File commands

Delete takes in one param, a file fully named and assumed to be in the present drive.

Rename takes in two params, the first the original full filename, and the next is the altered filename. Assumed to be the present drive.

Copy takes in two params. THe first params starts with 'driveLetter:filename.xxx' and the seconds follows the same structure.

Replace 'driveLetter' with the ascii character of the drive like you would in Windows, assuming that A and B are floppy drives and C-Z are HDD's.

Be sure to include the ':' next to the drive letter.

Size takes in one param, the fully named file assumed at the present drive.




Write.bin

Write is one of the types of executables that takes in param. It takes in the name of the file you want to edit.

Write is not like Windows Notepad. You do not have insert like you would in a normal text editor, you can only overwrite.

They are also formatted in CRLF, not CR, not LF.

You have access to 6 pages (80x25) to write on.

Press escape to leave the program without saving the file, and ~ to save the file and exit.

Basic.bin

Basic takes in a .tom file and runs it. You can create one in write. Or, you can make one on Windows and put in on a floppy disk and copy the file over.
